Avaliação#

  • Para saber se podemos confiar em nosso método ou sistema, precisamos avaliá-lo.

  • Seleção de modelos: escolha entre diferentes modelos de forma baseada em dados.

    • Se você não consegue medir, não consegue melhorar.

  • Convença os outros de que seu trabalho é significativo

    • Pares, liderança, clientes, você mesmo(!)

  • Quando possível, tente interpretar o que seu modelo aprendeu

    • O sinal que seu modelo encontrou pode ser apenas um artefato de seus dados tendenciosos

    • Veja ‘Por que devo confiar em você?’ por Marco Ribeiro et al.

Projetando sistemas de aprendizado de máquina#

  • Apenas executar seu algoritmo favorito geralmente não é uma ótima maneira de começar

  • Considere o problema: Como medir o sucesso? Existem custos envolvidos?

    • Você quer entender fenômenos ou fazer modelagem de caixa preta?

  • Analise os erros do seu modelo. Não faça ajustes finos indefinidamente.

    • Construir protótipos iniciais. Você deve coletar mais dados ou dados adicionais?

    • A tarefa deve ser reformulada?

  • Sistemas de aprendizado de máquina excessivamente complexos são difíceis de manter

    • Consulte ‘Aprendizado de máquina: o cartão de crédito com juros altos da dívida técnica’

Avaliações do mundo real#

  • Avalie as previsões, mas também como os resultados melhoram por causa delas

  • Cuidado com os ciclos de feedback: as previsões podem influenciar dados de entrada futuros

    • Recomendações médicas, filtragem de spam, algoritmos de negociação,…

  • Avalie algoritmos em estado selvagem.

    • Teste A/B: divida os usuários em grupos, teste diferentes modelos em paralelo

    • Teste de bandidos: direcione gradualmente mais usuários para o sistema vencedor

Técnicas de estimativa de desempenho#

  • Sempre avalie os modelos como se eles estivessem prevendo dados futuros

  • Não temos acesso a dados futuros, por isso fingimos que alguns dados estão ocultos

  • Maneira mais simples: o holdout (divisão simples de teste de trem)

    • Aleatoriamente dividir dados (e rótulos correspondentes) em conjunto de treinamento e teste (por exemplo, 75% -25%)

    • Treinar (ajustar) um modelo nos dados de treinamento, pontuar nos dados de teste

Validação cruzada K-fold#

  • Cada divisão aleatória pode produzir modelos (e pontuações) muito diferentes

    • por exemplo. todos os exemplos fáceis (ou difíceis) podem acabar no conjunto de testes

  • Divida os dados em k partes de tamanhos iguais, chamadas folds

    • Crie divisões k, cada vez usando uma dobra diferente como conjunto de teste

  • Calcular pontuações de avaliação k, agregar depois (por exemplo, calcular a média)

  • Examine a variação da pontuação para ver quão sensíveis (instáveis) são os modelos

  • Grande k fornece melhores estimativas (mais dados de treinamento), mas é caro

def plot_cv_indices(cv, X, y, group, ax, lw=2, show_groups=False, s=700, legend=True):
    """Create a sample plot for indices of a cross-validation object."""
    n_splits = cv.get_n_splits(X, y, group)

    # Generate the training/testing visualizations for each CV split
    for ii, (tr, tt) in enumerate(cv.split(X=X, y=y, groups=group)):
        # Fill in indices with the training/test groups
        indices = np.array([np.nan] * len(X))
        indices[tt] = 1
        indices[tr] = 0

        # Visualize the results
        ax.scatter([n_splits - ii - 1] * len(indices), range(len(indices)),
                   c=indices, marker='_', lw=lw, cmap=cmap_cv,
                   vmin=-.2, vmax=1.2, s=s)

    # Plot the data classes and groups at the end
    ax.scatter([-1] * len(X), range(len(X)),
               c=y, marker='_', lw=lw, cmap=cmap_data, s=s)
    yticklabels = ['class'] + list(range(1, n_splits + 1))

    if show_groups:
        ax.scatter([-2] * len(X), range(len(X)),
                   c=group, marker='_', lw=lw, cmap=cmap_group, s=s)
        yticklabels.insert(0, 'group')

    # Formatting
    ax.set(xticks=np.arange(-1 - show_groups, n_splits), xticklabels=yticklabels,
            ylabel='Sample index', xlabel="CV iteration",
            xlim=[-1.5 - show_groups, n_splits+.2], ylim=[-6, 100])
    ax.set_title('{}'.format(type(cv).__name__), fontsize=15)
    if legend:
        ax.legend([Patch(color=cmap_cv(.8)), Patch(color=cmap_cv(.2))],
                  ['Testing set', 'Training set'], loc=(1.02, .8))
    ax.spines['top'].set_visible(False)
    ax.spines['right'].set_visible(False)
    ax.spines['left'].set_visible(False)
    ax.spines['bottom'].set_visible(False)
    ax.set_yticks(())
    return ax

Validação cruzada estratificada K-Fold#

  • Se os dados estiverem desbalanceados, algumas classes terão apenas algumas amostras;

  • Provavelmente algumas classes não estão presentes no conjunto de testes;

  • Estratificação: proporções entre classes são conservadas em cada pasta

    • Ordenar exemplos por classe

    • Separar as amostras de cada classe em k conjuntos (estratos)

    • Combine os estratos correspondentes em dobras

Validação cruzada Leave-One-Out#

  • validação cruzada k fold com k igual ao número de amostras

  • Completamente imparcial (em termos de divisão de dados), mas computacionalmente caro

  • Na verdade generaliza menos bem para dados invisíveis

    • Os conjuntos de treinamento estão correlacionados (fortemente sobrepostos)

    • Overfits nos dados usados para (toda) a avaliação

    • Uma amostra diferente dos dados pode produzir resultados diferentes

  • Recomendado apenas para pequenos conjuntos de dados

O Bootstrap#

  • Amostra de pontos de dados n (tamanho do conjunto de dados), com substituição, como conjunto de treinamento (o bootstrap)

    • Em média, os bootstraps incluem 66% de todos os pontos de dados (alguns são duplicados)

  • Use as amostras sem amostragem (fora do bootstrap) como conjunto de teste

  • Repita \(k\) vezes para obter pontuações de \(k\)

  • Semelhante ao Shuffle-Split com train_size=0.66, test_size=0.34 mas sem duplicatas

from sklearn.utils import resample

# Toy implementation of bootstrapping
class Bootstrap:
    def __init__(self, nr):
        self.nr = nr

    def get_n_splits(self, X, y, groups=None):
        return self.nr

    def split(self, X, y, groups=None):
        indices = range(len(X))
        splits = []
        for i in range(self.nr):
            train = resample(indices, replace=True, n_samples=len(X), random_state=i)
            test = list(set(indices) - set(train))
            splits.append((train, test))
        return splits

Validação cruzada repetida#

  • A validação cruzada ainda é tendenciosa, pois a divisão inicial pode ser feita de várias maneiras

  • Validação cruzada repetida ou n vezes k-fold:

    • Embaralhe os dados aleatoriamente, faça validação cruzada k-fold

    • Repita n vezes, produz n vezes k pontuações

  • Imparcial, muito robusto, mas n vezes mais caro

Validação cruzada com grupos#

  • Às vezes, os dados contêm grupos inerentes:

    • Múltiplas amostras do mesmo paciente, imagens da mesma pessoa,…

  • Dados da mesma pessoa podem acabar no conjunto de treinamento e de teste

  • Queremos medir até que ponto o modelo se generaliza para outras pessoas

  • Certifique-se de que os dados de uma pessoa estejam no trem ou no conjunto de teste

    • Isso é chamado de agrupamento ou bloqueio

    • Validação cruzada de deixar um assunto de fora: conjunto de testes para cada assunto/grupo

Test-then-train (prequential evaluation)#

  • Every new sample is evaluated only once, then added to the training set

    • Can also be done in batches (of n samples at a time)

  • TimeSeriesSplit

    • In the kth split, the first k folds are the train set and the (k+1)th fold as the test set

    • Often, a maximum training set size (or window) is used

      • more robust against concept drift (change in data over time)

Testar e treinar (avaliação pré-quencial)#

  • Cada nova amostra é avaliada apenas uma vez e depois adicionada ao conjunto de treinamento

    • Também pode ser feito em lotes (de n amostras por vez)

  • TimeSeriesSplit

    • Na k-ésima divisão, as primeiras k dobras são o conjunto de trem e a (k+1)-ésima dobra como o conjunto de teste

    • Freqüentemente, um tamanho máximo de conjunto de treinamento (ou janela) é usado

      • mais robusto contra desvios de conceito (mudança nos dados ao longo do tempo)

Escolhendo um procedimento de estimativa de desempenho#

Não há regras rígidas, apenas diretrizes:

  • Sempre use estratificação para classificação (o sklearn faz isso por padrão)

  • Use validação para conjuntos de dados muito grandes (por exemplo,> 1.000.000 exemplos)

    • Ou quando os alunos nem sempre convergem (por exemplo, aprendizagem profunda)

  • Escolha k dependendo do tamanho e dos recursos do conjunto de dados

    • Use deixar um de fora para conjuntos de dados muito pequenos (por exemplo, <100 exemplos)

    • Use validação cruzada caso contrário

      • Mais popular (e teoricamente sólido): CV 10 vezes

      • A literatura sugere que um CV 5x2 vezes é melhor

  • Use agrupamento ou deixe um assunto de fora para dados agrupados

  • Use treinar e testar para séries temporais

Evaluation Metrics for Classification#

Avaliação vs Otimização#

  • Cada algoritmo otimiza uma determinada função objetivo (nos dados de treinamento)

    • Por exemplo. lembre-se da perda L2 na regressão Ridge $\(\mathcal{L}_{Ridge} = \sum_{n=1}^{N} (y_n-(\mathbf{w}\mathbf{x_n} + w_0))^2 + \alpha \sum_{i =0}^{p} w_i^2\)$

  • A escolha da função é limitada pelo que pode ser otimizado de forma eficiente

  • No entanto, avaliamos o modelo resultante com uma pontuação que faz sentido no mundo real

    • Porcentagem de previsões corretas (em um conjunto de testes)

    • O custo real dos erros (por exemplo, em dinheiro, tempo, vidas,…)

  • Também ajustamos os hiperparâmetros do algoritmo para maximizar essa pontuação

Classificação binária#

  • Temos uma classe positiva e uma negativa

  • 2 tipos diferentes de erros:

    • Falso Positivo (erro tipo I): o modelo prevê positivo enquanto o rótulo verdadeiro é negativo

    • Falso Negativo (erro tipo II): o modelo prevê negativo enquanto o rótulo verdadeiro é positivo

  • Nem sempre são igualmente importantes

    • De que lado você quer errar para um exame médico?

Matrizes de confusão#

  • Podemos representar todas as previsões (corretas e incorretas) em uma matriz de confusão

    • matriz n por n (n é o número de classes)

    • As linhas correspondem às classes verdadeiras, as colunas às classes previstas

    • Contar quantas vezes as amostras pertencentes a uma classe C são classificadas como C ou qualquer outra classe.

    • Para classificação binária, rotulamos estes verdadeiro negativo (TN), verdadeiro positivo (TP), falso negativo (FN), falso positivo (FP)

def plot_confusion_matrix(values, xlabel="predicted labels", ylabel="true labels", xticklabels=None,
                          yticklabels=None, cmap=None, vmin=None, vmax=None, ax=None,
                          fmt="{:.2f}", xtickrotation=45, norm=None, fsize=8):
    """Plot a matrix as heatmap with explicit numbers.
    Parameters
    ----------
    values : ndarray
        Two-dimensional array to visualize.
    xlabel : string, default=""
        Label for the x-axis.
    ylabel : string, default=""
        Label for the y-axis.
    xticklabels : list of string or None, default=None
        Tick labels for the x-axis.
    yticklabels : list of string or None, default=None
        Tick labels for the y-axis
    cmap : string or colormap
        Matpotlib colormap to use.
    vmin : int, float or None
        Minimum clipping value.
    vmax : int, float or None
        Maximum clipping value.
    ax : axes object or None
        Matplotlib axes object to plot into. If None, the current axes are
        used.
    fmt : string, default="{:.2f}"
        Format string to convert value to text.
    xtickrotation : float, default=45
        Rotation of the xticklabels.
    norm : matplotlib normalizer
        Normalizer passed to pcolor
    """
    import matplotlib.pyplot as plt
    plt.gcf().set_size_inches(fsize, fsize)
    if ax is None:
        ax = plt.gca()
    img = ax.pcolormesh(values, cmap=cmap, vmin=vmin, vmax=vmax, norm=norm)
    # this will allow us to access the pixel values:

Multi-class classification#

  • Train models per class : one class viewed as positive, other(s) als negative, then average

    • micro-averaging: count total TP, FP, TN, FN (every sample equally important)

      • micro-precision, micro-recall, micro-F1, accuracy are all the same $\(\text{Precision:} \frac{\sum_{c=1}^C\text{TP}_c}{\sum_{c=1}^C\text{TP}_c + \sum_{c=1}^C\text{FP}_c} \xrightarrow{c=2} \frac{\text{TP} + \text{TN}}{\text{TP} + \text{TN} + \text{FP} + \text{FN}}\)$

    • macro-averaging: average of scores \(R(y_c,\hat{y_c})\) obtained on each class

      • Preferable for imbalanced classes (if all classes are equally important)

      • macro-averaged recall is also called balanced accuracy $\(\frac{1}{C} \sum_{c=1}^C R(y_c,\hat{y_c})\)$

    • weighted averaging (\(w_c\): ratio of examples of class \(c\), aka support): \(\sum_{c=1}^C w_c R(y_c,\hat{y_c})\)

Classificação multiclasse#

  • Treinar modelos por classe: uma classe vista como positiva, outra(s) também negativa(s), depois média

    • micromédia: contagem total de TP, FP, TN, FN (cada amostra é igualmente importante)

      • microprecisão, micro-recall, micro-F1, precisão são todos iguais $\(\text{Precisão:} \frac{\sum_{c=1}^C\text{TP}_c}{\sum_{c=1}^C\text{TP}_c + \sum_{c=1 }^C\text{FP}_c} \xrightarrow{c=2} \frac{\text{TP} + \text{TN}}{\text{TP} + \text{TN} + \text{FP} + \text{FN}}\)$

    • macro-média: média das pontuações \(R(y_c,\hat{y_c})\) obtidas em cada classe

      • Preferível para classes desequilibradas (se todas as classes forem igualmente importantes)

      • o recall da média macro também é chamada de precisão balanceada $\(\frac{1}{C} \sum_{c=1}^C R(y_c,\hat{y_c})\)$

    • média ponderada (\(w_c\): proporção de exemplos da classe \(c\), também conhecido como suporte): \(\sum_{c=1}^C w_c R(y_c,\hat{y_c})\)

Outras métricas de classificação úteis#

  • Kappa de Cohen

    • Mede o ‘acordo’ entre diferentes modelos (também conhecido como acordo entre avaliadores)

    • Para avaliar um único modelo, compare-o com um modelo que faz suposições aleatórias

      • Semelhante à precisão, mas levando em consideração a possibilidade de prever a classe certa por acaso

    • Pode ser ponderado: diferentes classificações incorretas com pesos diferentes

    • 1: previsão perfeita, 0: previsão aleatória, negativo: pior que aleatório

    • Com \(p_0\) = precisão e \(p_e\) = precisão do classificador aleatório: $\(\kappa = \frac{p_o - p_e}{1 - p_e}\)$

  • Coeficiente de correlação de Matthews

    • Corrige dados desequilibrados, alternativa para precisão balanceada ou AUROC

    • 1: previsão perfeita, 0: previsão aleatória, -1: previsão inversa $\(MCC = \frac{tp \times tn - fp \times fn}{\sqrt{(tp + fp)(tp + fn)(tn + fp)(tn + fn)}}\)$

Avaliação probabilística#

  • Os classificadores muitas vezes podem fornecer estimativas de incerteza das previsões.

  • Lembre-se de que os modelos lineares, na verdade, retornam um valor numérico.

    • Quando \(\hat{y}<0\), preveja a classe -1, caso contrário, preveja a classe +1 $\(\hat{y} = w_0 * x_0 + w_1 * x_1 + ... + w_p * x_p + b \)$

  • Na prática, muitas vezes você está interessado em saber até que ponto um classificador está certo sobre a previsão de cada classe (por exemplo, tratamentos de câncer).

  • A maioria dos métodos de aprendizagem pode retornar pelo menos uma medida de confiança em suas previsões.

    • Função de decisão: valor de ponto flutuante para cada amostra (maior: mais confiável)

    • Probabilidade: probabilidade estimada para cada classe

A função de decisão#

No caso de classificação binária, o valor de retorno da função de decisão codifica o quão fortemente o modelo acredita em um ponto de dados pertence à classe “positiva”.

  • Valores positivos indicam preferência pela classe positiva.

  • O intervalo pode ser arbitrário e pode ser afetado por hiperparâmetros. Difícil de interpretar.

Precision-Recall curve#

  • The best trade-off between precision and recall depends on your application

    • You can have arbitrary high recall, but you often want reasonable precision, too.

  • Plotting precision against recall for all possible thresholds yields a precision-recall curve

    • Change the treshold until you find a sweet spot in the precision-recall trade-off

    • Often jagged at high thresholds, when there are few positive examples left

Curva recall - precisão#

  • A melhor relação entre precisão e recall depende da sua aplicação

    • Você pode ter um recall alto arbitrário, mas muitas vezes também deseja uma precisão razoável.

  • Traçar a precisão em relação ao recall para todos os limites possíveis produz uma curva recall - precisão

    • Altere o limite até encontrar um ponto ideal na compensação de recuperação de precisão

    • Frequentemente irregular em limites elevados, quando restam poucos exemplos positivos

Receiver Operating Characteristics (ROC)#

  • Trade off true positive rate \(\textit{TPR}= \frac{TP}{TP + FN}\) with false positive rate \(\textit{FPR} = \frac{FP}{FP + TN}\)

  • Plotting TPR against FPR for all possible thresholds yields a Receiver Operating Characteristics curve

    • Change the treshold until you find a sweet spot in the TPR-FPR trade-off

    • Lower thresholds yield higher TPR (recall), higher FPR, and vice versa

Características Operacionais do Receptor (ROC)#

  • Custo-benefício entre a taxa verdadeiros positivos \(\textit{TPR}= \frac{TP}{TP + FN}\) com taxa falsos positivos \(\textit{FPR} = \frac{FP}{FP + TN}\)

  • Traçar TPR contra FPR para todos os limites possíveis produz uma curva de características operacionais do receptor

    • Altere o limite até encontrar um ponto ideal na compensação TPR-FPR

    • Limites mais baixos geram TPR (recall) mais alto, FPR mais alto e vice-versa

    • Random guessing (TPR=FPR) or predicting majority class (TPR=FPR=1): 0.5 AUC

Multi-class AUROC (or AUPRC)#

  • We again need to choose between micro- or macro averaging TPR and FPR.

    • Micro-average if every sample is equally important (irrespective of class)

    • Macro-average if every class is equally important, especially for imbalanced data

AUROC multiclasse (ou AUPRC)#

  • Novamente precisamos escolher entre média micro ou macro TPR e FPR.

    • Micromédia se todas as amostras forem igualmente importantes (independentemente da classe)

    • Macromédia se todas as classes forem igualmente importantes, especialmente para dados desequilibrados

Model calibration#

  • For some models, the predicted uncertainty does not reflect the actual uncertainty

    • If a model is 90% sure that samples are positive, is it also 90% accurate on these?

  • A model is called calibrated if the reported uncertainty actually matches how correct it is

    • Overfitted models also tend to be over-confident

    • LogisticRegression models are well calibrated since they learn probabilities

    • SVMs are not well calibrated. Biased towards points close to the decision boundary.

Calibração do modelo#

  • Para alguns modelos, a incerteza prevista não reflete a incerteza real

  • Um modelo é chamado de calibrado se a incerteza relatada realmente corresponder ao quão correto ele é

    • Modelos overfitted também tendem a ser excessivamente confiantes

    • Os modelos LogisticRegression são bem calibrados, pois aprendem probabilidades *SVMs não estão bem calibrados. Intencionado em direção a pontos próximos ao limite de decisão.

Brier score#

  • You may want to select models based on how accurate the class confidences are.

  • The Brier score loss: squared loss between predicted probability \(\hat{p}\) and actual outcome \(y\)

    • Lower is better $\(\mathcal{L}_{Brier} = \frac{1}{n}\sum_{i=1}^n (\hat{p}_i - y_i)^2\)$

Pontuação de Brier#

  • Você pode selecionar modelos com base na precisão das confidências da classe.

  • A perda de pontuação de Brier: perda quadrada entre a probabilidade prevista \(\hat{p}\) e o resultado real \(y\)

    • Menor é melhor $\(\mathcal{L}_{Brier} = \frac{1}{n}\sum_{i=1}^n (\hat{p}_i - y_i)^2\)$

Isotonic regression#

  • Fit to minimize the loss between the uncalibrated predictions \(f(x)\) and the actual labels

Cost-sensitive classification (dealing with imbalance)#

  • In the real worlds, different kinds of misclassification can have different costs

    • Misclassifying certain classes can be more costly than others

    • Misclassifying certain samples can be more costly than others

  • Cost-sensitive resampling: resample (or reweight) the data to represent real-world expectations

    • oversample minority classes (or undersample majority) to ‘correct’ imbalance

    • increase weight of misclassified samples (e.g. in boosting)

    • decrease weight of misclassified (noisy) samples (e.g. in model compression)

Class weighting#

  • If some classes are more important than others, we can give them more weight

    • E.g. for imbalanced data, we can give more weight to minority classes

  • Most classification models can include it in their loss function and optimize for it

    • E.g. Logistic regression: add a class weight \(w_c\) in the log loss function $\(\mathcal{L_{log}}(\mathbf{w}) = - \sum_{c=1}^{C} \color{red}{w_c} \sum_{n=1}^{N} p_{n,c} log(q_{n,c}) \)$

Cost-sensitive algorithms#

  • Cost-sensitive algorithms

    • If misclassification cost of some classes is higher, we can give them higher weights

    • Some support cost matrix \(C\): costs \(c_{i,j}\) for every possible type of error

  • Cost-sensitive ensembles: convert cost-insensitive classifiers into cost-sensitive ones

    • MetaCost: Build a model (ensemble) to learn the class probabilities \(P(j|x)\)

      • Relabel training data to minimize expected cost: \(\underset{i}{\operatorname{argmin}} \sum_j P_j(x) c_{i,j}\)

      • Accuracy may decrease but cost decreases as well.

    • AdaCost: Boosting with reweighting instances to reduce costs

Tuning the decision threshold#

  • If every FP or FN has a certain cost, we can compute the total cost for a given model: $\(\text{total cost} = \text{FPR} * cost_{FP} * ratio_{pos} + (1-\text{TPR}) * cost_{FN} * (1-ratio_{pos})\)$

  • This yields different isometrics (lines of equal cost) in ROC space

  • Optimal threshold is the point on the ROC curve where cost is minimal (line search)

Regression metrics#

Most commonly used are

  • mean squared error: \(\frac{\sum_{i}(y_{pred_i}-y_{actual_i})^2}{n}\)

    • root mean squared error (RMSE) often used as well

  • mean absolute error: \(\frac{\sum_{i}|y_{pred_i}-y_{actual_i}|}{n}\)

    • Less sensitive to outliers and large errors

R squared#

  • \(R^2 = 1 - \frac{\color{blue}{\sum_{i}(y_{pred_i}-y_{actual_i})^2}}{\color{red}{\sum_{i}(y_{mean}-y_{actual_i})^2}}\)

    • Ratio of variation explained by the model / total variation

    • Between 0 and 1, but negative if the model is worse than just predicting the mean

    • Easier to interpret (higher is better).

Bias-Variance decomposition#

  • Evaluate the same algorithm multiple times on different random samples of the data

  • Two types of errors can be observed:

    • Bias error: systematic error, independent of the training sample

      • These points are predicted (equally) wrong every time

    • Variance error: error due to variability of the model w.r.t. the training sample

      • These points are sometimes predicted accurately, sometimes inaccurately

Decomposição de polarização-variância#

  • Avalie o mesmo algoritmo várias vezes em diferentes amostras aleatórias de dados

  • Dois tipos de erros podem ser observados:

    • Erro de viés: erro sistemático, independente da amostra de treinamento

      • Esses pontos são previstos (igualmente) errados todas as vezes

    • Erro de variância: erro devido à variabilidade do modelo em relação ao valor. a amostra de treinamento

      • Esses pontos às vezes são previstos com precisão, às vezes de forma imprecisa

Computing bias and variance error#

  • Take 100 or more bootstraps (or shuffle-splits)

  • Regression: for each data point x:

    • \(bias(x)^2 = (x_{true} - mean(x_{predicted}))^2\)

    • \(variance(x) = var(x_{predicted})\)

  • Classification: for each data point x:

    • \(bias(x)\) = misclassification ratio

    • \(variance(x) = (1 - (P(class_1)^2 + P(class_2)^2))/2\)

      • \(P(class_i)\) is ratio of class \(i\) predictions

  • Total bias: \(\sum_{x} bias(x)^2 * w_x\) \(w_x\): the percentage of times \(x\) occurs in the test sets

  • Total variance: \(\sum_{x} variance(x) * w_x\)

Viés de cálculo e erro de variância#

  • Pegue 100 ou mais bootstraps (ou divisões aleatórias)

  • Regressão: para cada ponto de dados x:

    • \(viés(x)^2 = (x_{verdadeiro} - média(x_{previsto}))^2\)

    • \(variância(x) = var(x_{previsto})\)

  • Classificação: para cada ponto de dados x:

    • \(viés(x)\) = taxa de classificação incorreta

    • \(variância(x) = (1 - (P(class_1)^2 + P(class_2)^2))/2\)

      • \(P(class_i)\) é a proporção das previsões da classe \(i\)

  • Viés total: \(\sum_{x} preconceito(x)^2 * w_x\) \(w_x\): a porcentagem de vezes que \(x\) ocorre nos conjuntos de teste

  • Variância total: \(\sum_{x} variância(x) * w_x\)

from sklearn.model_selection import ShuffleSplit, train_test_split, GridSearchCV

# Bias-Variance Computation
def compute_bias_variance(clf, X, y):
    # Bootstraps
    n_repeat = 40 # 40 is on the low side to get a good estimate. 100 is better.
    shuffle_split = ShuffleSplit(test_size=0.33, n_splits=n_repeat, random_state=0)

    # Store sample predictions
    y_all_pred = [[] for _ in range(len(y))]

    # Train classifier on each bootstrap and score predictions
    for i, (train_index, test_index) in enumerate(shuffle_split.split(X)):
        # Train and predict
        clf.fit(X[train_index], y[train_index])
        y_pred = clf.predict(X[test_index])

        # Store predictions
        for j,index in enumerate(test_index):
            y_all_pred[index].append(y_pred[j])

    # Compute bias, variance, error
    bias_sq = sum([ (1 - x.count(y[i])/len(x))**2 * len(x)/n_repeat
                for i,x in enumerate(y_all_pred)])
    var = sum([((1 - ((x.count(0)/len(x))**2 + (x.count(1)/len(x))**2))/2) * len(x)/n_repeat
               for i,x in enumerate(y_all_pred)])
    error = sum([ (1 - x.count(y[i])/len(x)) * len(x)/n_repeat
            for i,x in enumerate(y_all_pred)])

    return np.sqrt(bias_sq), var, error

Bias and variance, underfitting and overfitting#

  • High variance means that you are likely overfitting

    • Use more regularization or use a simpler model

  • High bias means that you are likely underfitting

    • Do less regularization or use a more flexible/complex model

  • Ensembling techniques (see later) reduce bias or variance directly

    • Bagging (e.g. RandomForests) reduces variance, Boosting reduces bias

Understanding under- and overfitting#

  • Regularization reduces variance error (increases stability of predictions)

    • But too much increases bias error (inability to learn ‘harder’ points)

  • High regularization (left side): Underfitting, high bias error, low variance error

    • High training error and high test error

  • Low regularization (right side): Overfitting, low bias error, high variance error

    • Low training error and higher test error

Hyperparameter tuning#

  • There exists a huge range of techniques to tune hyperparameters. The simplest:

    • Grid search: Choose a range of values for every hyperparameter, try every combination

      • Doesn’t scale to many hyperparameters (combinatorial explosion)

    • Random search: Choose random values for all hyperparameters, iterate \(n\) times

      • Better, especially when some hyperparameters are less important

  • Many more advanced techniques exist, see lecture on Automated Machine Learning

  • First, split the data in training and test sets (outer split)

  • Split up the training data again (inner cross-validation)

    • Generate hyperparameter configurations (e.g. random/grid search)

    • Evaluate all configurations on all inner splits, select the best one (on average)

  • Retrain best configurations on full training set, evaluate on held-out test data

Nested cross-validation#

  • Simplest approach: single outer split and single inner split (shown below)

  • Risk of over-tuning hyperparameters on specific train-test split

    • Only recommended for very large datasets

  • Nested cross-validation:

    • Outer loop: split full dataset in \(k_1\) training and test splits

    • Inner loop: split training data into \(k_2\) train and validation sets

  • This yields \(k_1\) scores for \(k_1\) possibly different hyperparameter settings

    • Average score is the expected performance of the tuned model

  • To use the model in practice, retune on the entire dataset

Summary#

  • Split the data into training and test sets according to the application

    • Holdout only for large datasets, cross-validation for smaller ones

    • For classification, always use stratification

    • Grouped or ordered data requires special splitting

  • Choose a metric that fits your application

    • E.g. precision to avoid false positives, recall to avoid false negatives

  • Calibrate the decision threshold to fit your application

    • ROC curves or Precision-Recall curves can help to find a good tradeoff

  • If possible, include the actual or relative costs of misclassifications

    • Class weighting, instance weighting, ROC isometrics can help

    • Be careful with imbalanced or unrepresentative datasets

  • When using the predicted probabilities in applications, calibrate the models

  • Always tune the most important hyperparameters

    • Manual tuning: Use insight and train-test scores for guidance

    • Hyperparameter optimization: be careful not to over-tune

Resumo#

  • Divida os dados em conjuntos de treinamento e teste de acordo com a aplicação

    • Holdout apenas para conjuntos de dados grandes, validação cruzada para conjuntos menores *Para classificação, use sempre estratificação

    • Dados agrupados ou ordenados requerem divisão especial

  • Escolha uma métrica adequada à sua aplicação

    • Por exemplo. precisão para evitar falsos positivos, recall para evitar falsos negativos

  • Calibre o limite de decisão para se adequar à sua aplicação

    • Curvas ROC ou curvas Precision-Recall podem ajudar a encontrar uma boa compensação

  • Se possível, inclua os custos reais ou relativos de classificações incorretas

    • Ponderação de classe, ponderação de instância e isometria ROC podem ajudar

    • Tenha cuidado com conjuntos de dados desequilibrados ou não representativos

  • Ao usar as probabilidades previstas em aplicativos, calibre os modelos

  • Sempre ajuste os hiperparâmetros mais importantes

    • Ajuste manual: use insights e pontuações de testes de treinamento para orientação

    • Otimização de hiperparâmetros: tome cuidado para não ajustar demais
